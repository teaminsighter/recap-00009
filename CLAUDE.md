This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

VFS Slot Checker - Tools for checking VFS Global visa appointment availability. Contains:
- **Chrome Extension** (Manifest V3) - Full-featured slot checker with API integration
- **Userscripts** (Tampermonkey) - Lightweight page watchers with auto-fill

## Development Commands

**Load Chrome extension for testing:**
1. Navigate to `chrome://extensions/`
2. Enable "Developer mode"
3. Click "Load unpacked"
4. Select the `extracted_extension/` directory

**Package extension for distribution:**
```bash
zip -r vfs-slot-checker.zip extracted_extension/
```

**Install userscripts:**
1. Install Tampermonkey browser extension
2. Open `vfs-slot-watcher.user.js` or `vfs-helper.user.js`
3. Tampermonkey will prompt to install

No build, lint, or test commands - this is vanilla JS with no tooling.

## Architecture

### Chrome Extension

#### Component Communication Flow

```
VFS Website
    ↓
Content Script (content.js)
    ├─ Injects fetch/XHR interceptor into page context
    ├─ Captures RSA keys, auth tokens, email from sessionStorage
    └─ Sends credentials to Background via chrome.runtime.sendMessage
    ↓
Background Service Worker (background.js)
    ├─ Stores credentials in chrome.storage.local
    ├─ Manages RSA-OAEP encryption with Web Crypto API
    ├─ Makes authenticated API calls to lift-api.vfsglobal.com
    └─ Handles auto-check alarms
    ↓
Popup UI (popup.js + popup.html)
    ├─ Displays status, calendar, time slots
    └─ Manages booking workflow (OTP → Date → Time → Book)
```

#### Key Files

| File | Purpose |
|------|---------|
| `manifest.json` | Extension config, permissions, entry points |
| `background.js` | Service worker: API calls, RSA encryption, state management |
| `content.js` | Page injection: credential capture, request interception |
| `popup.js` | UI logic: calendar, slots, booking workflow |
| `popup.html` | UI markup with embedded CSS |

#### VFS API Endpoints (lift-api.vfsglobal.com)

- `POST /appointment/slots` - Check available slots
- `POST /appointment/calendar` - Get available dates
- `POST /appointment/timeslot` - Get time slots for a date
- `POST /appointment/applicantotp` - Send/verify OTP
- `POST /appointment/schedule` - Book appointment
- `POST /appointment/application` - Get application status
- `POST /appointment/fees` - Get fees info

#### State Management

**Chrome Storage (persistent):**
```javascript
vfsState = {
  rsaKey,           // Base64 RSA public key from sessionStorage
  authorize,        // Bearer token captured from requests
  email,            // User email
  route,            // Country/mission route header
  interval,         // Auto-check interval in seconds
  capturedPayload   // Latest captured request payload
}
```

**In-Memory (background.js):**
- `cryptoKey` - Web Crypto CryptoKey object for RSA-OAEP
- `ready` - Boolean indicating all credentials are present

#### Token Generation

The `clientsource` header is generated by RSA-OAEP encrypting `email + UTC timestamp` (format: `YYYY-MM-DDTHH:MM:SS`) using the captured RSA public key.

### Userscripts

Two standalone Tampermonkey scripts that run directly on VFS pages:

| File | Purpose |
|------|---------|
| `vfs-slot-watcher.user.js` | Full-featured watcher with tabs, form data management |
| `vfs-helper.user.js` | Simpler version with same core functionality |

Both scripts:
- Create draggable floating UI panel on VFS pages
- Monitor page for slot availability keywords
- Auto-fill form fields using `fieldMappings` object
- Store form data in `localStorage`
- Play audio alerts when slots detected
- Support configurable refresh intervals

## Code Conventions

- `$()` is a shortcut for `document.getElementById()` in popup.js
- Message handlers return `true` to indicate async response
- Section headers use decorative comment borders (`// ════════`)
- All CSS is embedded in popup.html `<style>` tag
- Userscripts use IIFE pattern with `'use strict'`
